# M5.2 — Scalar Broadcasting in fn-body BinOp

**Crate:** `malus-sema` (type rules); `malus-codegen-gpu` (broadcast kernel synth); `malus-codegen-cpu` (lowering)
**Done when:** `fn scale(a: Tensor<f32>, factor: f32) -> Tensor<f32> { return a * factor }` compiles and runs correctly, dispatching to a built-in `malus_mul_scalar` kernel.

## Background

M5.1 delivered built-in element-wise kernels for same-shape tensor BinOps (`a + b`). Scalar broadcasting (`a * factor` where one operand is a tensor and the other is a scalar) was explicitly out of scope. The `import_demo/ops.ml::scale` example exercises this path and currently fails at sema with a type mismatch.

## Scope

- Relax `check_binop` to allow one tensor + one scalar operand (same dtype) when the op is arithmetic (`Add`/`Sub`/`Mul`/`Div`).
- Synthesize `malus_mul_scalar` / `malus_add_scalar` / etc. MSL kernels (or a single parameterized kernel with a broadcast flag).
- Lower in codegen-cpu: detect tensor×scalar BinOp, dispatch to the scalar-broadcast built-in.

## Out of scope

- NumPy-style shape broadcasting (e.g. `[3,1] * [1,4]` → `[3,4]`) — full rank broadcasting is v1.
- Comparison ops with scalar broadcasting.
